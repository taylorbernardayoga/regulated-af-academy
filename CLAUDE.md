# Regulated AF Academy

This is the project folder for the Regulated AF Academy website. Read the files below before building or editing anything.

## Read these first, in order

1. `BRAND.md`: who this is for, the transformation, the identity, voice, and offer. The why.
2. `DESIGN.md`: the frontend design system. Colors, type, components, the look. The how.
3. `styles.css`: those design decisions as real code. Link it in every page.

## The offer in one line

Regulated AF Academy: a 4-month 1:1 coaching container for self-aware women who spiral, shut down, or sabotage relationships and want to move from anxious to secure. $2,200, or 4 payments of $597. 12 calls (3 per month, 50 minutes each) plus dripped education modules.

## Voice rules (critical)

- Never use em-dashes anywhere. Use commas, periods, or restructure the sentence. Check before finishing.
- Warm, direct, grounded, bold. Spicy identity-naming copy ("oh fuck, that's me"). Swearing is on-brand.
- Sentence case, plain verbs, no filler, no corporate wellness speak, no toxic positivity.

## Tech conventions

- Static marketing pages, plain HTML and CSS.
- New pages link `styles.css` for the shared design system, then add page-specific styles.
- The Academy CTA is always: book a call.

## Guardrails

- Confirm before changing any deployment, DNS, or domain settings.
- Match the existing color and type system unless asked to change it.
- Always check copy for em-dashes before finishing.

## File map

- `BRAND.md`: brand and messaging source of truth
- `DESIGN.md`: frontend design system map
- `styles.css`: shared design tokens and components
- `index.html`: Regulated AF Academy landing page
